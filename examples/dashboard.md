Use the Brand-Led UI Designer perspective.

Project:
A dashboard for a finance / crypto product.

Goal:
Help users quickly scan key information, understand current status, and take primary actions without the interface feeling like a generic exchange dashboard.

Platform:
Responsive web

This should feel:
- calm
- premium
- sharp
- information-dense but controlled
- brand-aware

Closer to:
- Linear precision
- Robinhood restraint
- editorial layout discipline

Avoid:
- generic crypto dashboard aesthetics
- too many boxed modules
- excessive borders and dividers
- loud color coding everywhere
- blue-purple tech gradients
- overbuilt enterprise UI
- every metric competing equally

What must stand out:
- the most important account / market / portfolio information
- the user’s next likely action
- one clear reading order across the screen

What should stay quiet:
- secondary metrics
- supporting modules
- less important filters / actions
- structural UI chrome

Desired density:
- balanced to dense, but calm

Constraints:
- Needs to support real data density
- Must include summary modules, data views, and primary action areas
- Must remain usable on smaller laptop screens
- Mobile adaptation should preserve hierarchy, not just stack everything
- Accessibility should meet WCAG AA

What I need from you:
Give me one strong dashboard direction, including:
1. Visual intent
2. Why this avoids generic dashboard patterns
3. Hierarchy strategy across the full screen
4. Brand expression beyond accent color
5. UI foundations
6. Recommendations for modules / surfaces / data presentation
7. What should be removed, merged, or visually quieted
8. Anti-generic review

Important:
Do not solve hierarchy by adding more cards or separators.
Use typography, spacing, and composition first.
